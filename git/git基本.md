バージョン管理を行うツール  
ソースコードの履歴確認  
自分のパソコンで完結する  
git init　初期化　　
- 新しいリポジトリ作成の際に使う
- gitによって管理されるようになる（作業ディレクトリになる）  
- 作業したいディレクトリに移動して`git init`実行

git status
- 現在のリポジトリの確認

git add
- 作業ディレクトリ内で変更されたファイルをステージングエリアに追加する　　
- `git add ファイル名`で特定のファイルのみ、`git add .`すべてのファイル

git commit(重要)
- ステージングエリアに追加された変更をリポジトリに記録　　
- `git commit -m "comment"`
- コメントはわかりやすく

git log　　
- コミット履歴の確認

　git branch
- 確認と新しいブランチの作成
- `git branch ブランチ名`これで新しいブランチ
- 実験的に使うことができ、ほかの作業との平行、修正がしやすい

git checkout  
- ブランチの切り替え  
- `git checkout ブランチ名`

　
git merge  
- 作業していたブランチの結合  
- まず統合されるブランチに移動して`git maerge 統合したいブランチ`

git push  
- ローカルリポジトリの変更をリモートリポジトリに送信するのに使用  

git pull  
- リモートリポジトリの最新の変更をローカルリポジトリに統合する
- ほかのメンバーが変更したリモートリポジトリに変更を加えた際最新の状態を自分のローカルに反映するため

## git&githubでの基本フロー  
リモートリポジトリの作成  
↓  
作業  
↓  
ステージング(add):履歴に登録する準備  
↓  
コミット(commit)：変更履歴を記録する  
↓  
プッシュ(push):ローカルリポジトリのコミットをリモートリポジトリに送信する
